
## 피드백 진단 (틔움 김선길 대표)

**핵심 1줄**: "부모리포트는 해도 그만 안해도 그만. 원장이 돈 내고 쓸 이유는 **행정 고통 해방**이다."

### 정확히 맞은 부분 (반박 불가)
- **부모는 케어플/바우처리 모르고 관심 없음** → 부모리포트 단독으로는 케어플 갈아타기 트리거 안됨
- **케어플 락인의 본질** = 끄면 센터 마비. 우리도 그 수준의 운영 의존성 만들어야 함
- **바우처 일지** = 6개월치 몰아쓰는 진짜 페인. "사진 한 장 → AI 풀어쓰기" 정확히 우리가 만든 흐름
- **지도점검(연 1회 감사)** = 원장이 1억이라도 낼 의향 있는 최상위 페인
- **케어플 마이그레이션 부재** = 초기 셋팅 이관 도구 없으면 갈아탈 수 없음 (현재 우리 치명적 공백)

### 우리가 놓치고 있던 포지셔닝 오류
현재 `/b2b-center` 랜딩 + Wave 1이 "수납·바우처청구·온보딩" 3개 있지만, **세일즈 메시지가 여전히 "부모 월간리포트 차별화"** 로 나가고 있음. 대표가 사이트 보고 "갈아탈 이유 설득 안 됨"이라고 한 게 이거. 코드는 절반 와있는데 **랜딩·내비·온보딩 wedge가 부모리포트로 잘못 박혀있는 상태**.

---

## 전략 재정렬: 부모리포트는 Phase 2 미끼, Phase 1은 "원장 ERP"

```text
[현재] 부모리포트 = Wedge → 원장 ERP = 부수
[변경] 원장 ERP(행정자동화) = Wedge → 부모리포트 = 락인/광고 수익화 Phase 2
```

가격(₩39,000/월)·B2B→B2C→B2G 풀스택 동선은 그대로. 단지 **첫 갈고리(wedge)** 를 부모용에서 원장용으로 교체.

---

## Phase 1 (지금 ~ 베타 종료): "원장이 ₩39,000 결제할 4대 이유"

대표가 명시적으로 꼽은 4가지 + 마이그레이션 1개. 모두 코드베이스에서 부분 구현 또는 메모리 상 합의 완료된 항목이라 **신규 발명 없이 완성도/위치만 끌어올리면 됨**.

### 1. 바우처 일지 자동화 (이미 골조 있음 — 완성도 90% → 100%)
- 현재: `analyze-session-upload` 엣지(Gemini Vision OCR) + `generate-weekly-therapy-note` 존재
- **추가할 것**:
  - "한 줄 메모 → AI 장문 일지" 모드 (사진뿐 아니라 텍스트 1줄 입력으로도 풀어쓰기)
  - **출력 → A4 인쇄 → 철** 워크플로우 (바우처 양식 호환 PDF, 자동 페이지네이션, 도장란)
  - "6개월치 몰아쓰기" 대응: 일자별 백필 모드 (날짜 지정 + 일괄 생성)
- 위치: `/b2b-center/app/intelligence/therapy-notes` 강화

### 2. 지도점검(감사) 대응 키트 (이미 `voucher_audit_*` 테이블 4개 존재 — UI/AI만 보강)
- 현재: `VoucherAuditPage` placeholder + `audit_checklist_items`, `voucher_audit_categories/items/requirements` 테이블 ready
- **추가할 것**:
  - 지도점검 체크리스트 (시·도별 최신 기준 큐레이션, 카드형 진행률)
  - **누락 서류 자동 진단** → AI가 누락된 행정서류 양식 자동 생성 (출석부, 동의서, 일지 누락분 등)
  - 지도점검 D-30 알림 + 1주 단위 준비 가이드
- 위치: `/b2b-center/app/audit` 신규 활성화

### 3. 수납 자동화 (이미 `BillingStatsPage` + `center_billing_closings` 존재 — 발송 액션 추가)
- **추가할 것**:
  - **매달 자동 수납 안내 문자/카톡** (본인부담금 + 가상계좌 + 입금확인 링크)
  - 입금 대조 자동화(은행 SMS 파싱 또는 수동 매칭)
  - 미수금 N일차 자동 독촉 룰 엔진
- 위치: 기존 `BillingStatsPage` 확장 + 메시징 모듈 신규

### 4. 통신 허브 (전화/문자/메모) — **신규**
- 대표 요청 가장 임팩트 큰 항목: "**전화 오면 학부모 번호가 사이트에 떠서 메모/녹음**"
- 1차 현실 가능 범위:
  - **착신번호 검색 1초 위젯** (전화벨 울릴 때 원장이 폰 번호 입력 → 학부모/아동 즉시 카드 표출, 최근 일지·미수금·다음 회기 한눈에)
  - 클릭 한 번으로 음성 메모 녹음 → 자동 STT → 해당 아동 타임라인 첨부
  - 보낸 문자 이력 자동 기록 (외부 발송 후 결과 수기 입력 or 카톡 알림톡 연동)
- 통화녹음 자동 연동(폰 OS 제약)은 Phase 2 (안드로이드 Capacitor 접근 권한 별도)

### 5. 케어플 마이그레이션 마법사 — **신규, 가장 시급**
- 대표 핵심 지적: "**초기 셋팅이 그대로 와야 갈아탐**"
- 케어플에서 추출 가능한 데이터(CSV 내보내기·스크린샷·관리자 화면 캡쳐) 받아서 → 우리 스키마(`center_clients`, `center_therapists`, `center_sessions`, `center_vouchers`)로 자동 매핑
- 진행 흐름: 케어플 데이터 업로드 → AI가 컬럼 매핑 추론 → 미리보기 → 일괄 import (`center_import_jobs` 테이블 이미 존재)
- 위치: `/b2b-center/app/setup` 온보딩 위저드에 "케어플에서 이전" 트랙 추가

### 6. AI 운영 가이드 챗봇 — **신규, 비용 대비 효과 최상**
- 대표 명시: "**원장 단톡방에서 매일 바우처 변경/신청 어떻게 하나요? 물어봄**"
- 사이드바 우하단 floating: 바우처/지도점검/행정 FAQ를 Gemini 3.1로 답변
- 지식베이스: 보건복지부·시도 공고문·바우처 매뉴얼 크롤링 (`docs/` 또는 신규 `center_knowledge_base` 테이블)
- 위치: B2B Center 콘솔 전역 floating (`copilotFlows.ts` 패턴 재사용)

### Phase 1 비-우선순위 (의도적 보류)
- ❌ 부모 월간/주간 리포트 신기능 (현재 수준 유지, 영업 메시지에서 빼기)
- ❌ 화이트라벨 정교화 (방금 만들었지만 보조 기능으로 강등)
- ❌ 스토어/광고 수익화

## Phase 2 (Phase 1 KPI 트리거 후): 부모 락인 + 광고 수익

진입 조건 (대표가 "쓸만하다" 인정할 KPI):
- 유료 센터 N개 도달 + 평균 월 활성 사용일 X일 + 케어플 마이그레이션 성공 사례 Y건
- 그때 부모리포트 + 부모앱 + 광고 노출을 본격화 (현재 메모리의 키즈노트 락인 전략 그대로)

## 기술 메모

- 모든 신규 테이블(`center_knowledge_base` 등)은 메모리의 GRANT 규칙·RLS 규칙 준수
- AI 호출은 Gemini 3.1 (`google/gemini-3-flash-preview` 또는 reasoning medium) — 메모리 표준
- 가격(₩39,000/월)·SKU(`b2b-center` plan)는 DB/코드 상수에서 읽기 — 신규 하드코딩 금지
- 메시징 발송(SMS/알림톡)은 외부 게이트웨이 필요 — Aligo·NHN Toast 중 선택 필요 (별도 시크릿)
