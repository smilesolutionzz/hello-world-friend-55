
좋은 방향입니다. `/mind-track`을 "결제 후 진짜 가치가 작동하는 곳"으로 만드는 통합 플랜을 정리할게요.

## 현재 진단

먼저 `/mind-track`이 지금 어떤 상태인지 빠르게 확인이 필요합니다 (실제 코드 본 후 확정):
- 페이지가 존재하나? 어떤 데이터를 표시하나?
- 결제 후 어디로 랜딩하나? mind-track인가 다른 곳인가?
- 데이터 누적(검사·관찰·금쪽·코칭) → mind-track 표시 연결 상태?
- 리포트 자동 생성/이메일 전송 파이프라인 존재?

## 목표 사용자 플로우 (결제 후 → 가치 실현)

```
[결제 완료]
    ↓
[1] /mind-track 자동 랜딩 + 환영 온보딩 4스텝
    "축하해요 — 이제 모든 데이터가 한곳에 모입니다"
    ↓
[2] 첫 데이터 입력 가이드 (체크리스트)
    □ 첫 검사 받기 (1분) → 자동 mind-track에 카드로 적재
    □ 첫 관찰일지 (30초)
    □ 금쪽상담소 1세션 (3분)
    ↓
[3] 데이터 ≥3건 시 자동 트리거
    "📊 종합 리포트 생성 준비 완료" 배너
    → [한 번에 만들기] 클릭 → /report-generator-pro 자동 선택 + 생성
    ↓
[4] 리포트 완성 시
    - mind-track 타임라인에 카드로 표시
    - 자동으로 이메일 전송 (PDF 첨부 대신 보안 링크)
    - "30일 후 변화 추적 리포트가 자동 생성됩니다" 예약
    ↓
[5] 30·60일 후 자동 재생성 + 이메일 알림 (RCI 기반 변화 추적)
```

## 구현 범위 (5개 모듈)

### 모듈 1. /mind-track 페이지 리뉴얼
- 결제 후 첫 진입 시 **환영 온보딩 모달** (4스텝, localStorage 1회만)
- **데이터 누적 카운터** 카드: 검사 N개 / 관찰 N개 / 금쪽 N개 / 코칭 N개
- **타임라인**: 모든 활동을 시간순 카드로 (이미 있는 timeline_activities 활용)
- **다음 액션 추천 카드** 3개: "검사 더 하기", "리포트 만들기", "전문가 매칭"

### 모듈 2. 결제 후 랜딩 통일
- `PostPaymentConversionFlow` / 결제 success 콜백 → `/mind-track?welcome=1` 자동 이동
- 기존 진입점 모두 mind-track 경유로 정리

### 모듈 3. "리포트 만들 타이밍" 자동 트리거
- 데이터 ≥3건 누적 시 mind-track 상단에 큰 CTA 배너
- 이미 만든 `ReportHubReadyBanner` 재사용 + mind-track 페이지에 mount

### 모듈 4. 리포트 자동 이메일 전송
- 새 DB 테이블: `report_email_subscriptions` (사용자별 자동 전송 on/off, 주기, 대상 이메일)
- 새 edge function: `send-report-email` — 리포트 생성 완료 시 호출되어 이메일 전송
- `email_domain--scaffold_transactional_email` (Lovable 내장 이메일) 사용
- 새 템플릿: `report-ready-notification.tsx` (리포트 완성 알림 + 보안 링크)

### 모듈 5. 30·60일 자동 재생성 (선택, 후속)
- pg_cron + edge function `auto-generate-followup-report`
- 첫 리포트 생성 시점 + 30/60일 시점에 데이터 재집계 → 자동 리포트 → 이메일

## 진행 순서 제안

1. **현재 /mind-track 코드 확인** (5분)
2. **모듈 1 + 2 + 3** (페이지 리뉴얼 + 진입 통일 + 트리거) — 이번 1차
3. **이메일 인프라 셋업 다이얼로그** → 도메인 확인 — 1차 끝나고
4. **모듈 4** (자동 이메일) — 2차
5. **모듈 5** (30·60일 재생성) — 3차 (안정화 후)
