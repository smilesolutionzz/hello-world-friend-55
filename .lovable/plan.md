# "무료 고민 리포트" → "내 트랙 찾기" 리포지셔닝

## 의도

현재 `/mind-track`의 "무료 고민 리포트 받기" 카드는 결과가 길고(요약·상태바·원인·즉시액션·추천트랙 5개 카드) 결제 동선과 분리돼 보임. 사용자가 원하는 건 "내 고민 → 짧은 진단 → 맞는 트랙 추천"이므로 카드 자체를 **트랙 추천 위저드**로 재포지셔닝하고 리포트는 보조로 축소.

엣지 함수(`mind-track-concern-report`)는 **그대로** 사용 — 프론트 표시 방식만 바꿉니다.

## 1. 입력 카드 리브랜딩

- 제목: "무료 고민 리포트 받기" → **"내 트랙 찾기"**
- 서브카피: "1분이면 돼요. 고민을 적어주시면 30일 트랙을 골라드려요."
- 버튼: "무료 리포트 받기" → **"내 트랙 찾기"**
- 작은 배지: "무료 · 비로그인 OK" 유지
- Textarea/AI 다듬기 그대로

## 2. 결과 섹션 축소 — 짧은 리포트

기존 5개 큰 카드 → 한 장의 컴팩트 카드로 통합:

```text
┌─────────────────────────────────────┐
│ 지금 마음 들여다보기                 │
│ {summary 2~3줄}                     │
│                                     │
│ ─ 현재 상태 ─                       │
│ 스트레스 ▮▮▮▮▯  72                 │
│ 에너지   ▮▮▯▯▯  38                 │
│ 명료함   ▮▮▮▯▯  55                 │
│                                     │
│ [짚어볼 만한 원인 보기 ▾] (접힘)    │
│ [오늘 바로 해볼 것 보기 ▾] (접힘)   │
└─────────────────────────────────────┘
```

`rootCauses`·`quickActions`는 Accordion으로 접어 노이즈 제거.

## 3. 트랙 추천 — TOP 3 카드

리포트 카드 바로 아래 헤드라인 **"이 고민에 맞는 트랙 3가지"**:

```text
┌──── 1순위 (가장 잘 맞음) ────┐
│ 🌿 스트레스 다스리기          │
│ "긴장 신호 인식 → 호흡 루틴…"  │
│ ✓ 매칭 이유 1줄              │
│ [이 트랙 선택]               │
└──────────────────────────────┘

┌──── 2순위 ────┐  ┌──── 3순위 ────┐
│ 🌙 수면 회복   │  │ ☀️ 감정 안정  │
│ [선택]        │  │ [선택]       │
└───────────────┘  └──────────────┘
```

생성 로직(클라이언트 사이드, 신규 백엔드 X):
- 1순위: 엣지 함수가 반환한 `trackRecommendation.matchedGoal`
- 2~3순위: `mindTrackCategories.ts`의 `recommendTracks()`에 `riskConcerns = TRACK_TAGS[matchedGoal].concern`을 넣고 점수 상위 트랙 중 1순위 제외하고 2개

선택 시 `setSelectedGoal(id)` → 기존 하단 sticky CTA 바가 자동으로 그 트랙·가격·시작 버튼을 노출 (현재 구조 그대로 활용).

기존 "이 고민에 딱 맞는 30일 트랙" 단일 카드 + 가격 박스는 **제거** (sticky CTA가 대체).

## 4. 흐름 연결

- 결과 받은 직후 `concern-report-result` 영역으로 자동 스크롤(현재 동작 유지)
- TOP3 중 하나 클릭 → sticky CTA 노출 + 부드럽게 "결제 진행은 하단의 시작 버튼에서" 토스트 1회
- "다른 트랙도 보기" 보조 링크 → 기존 `goal-section`의 advanced disclosure 열고 스크롤
