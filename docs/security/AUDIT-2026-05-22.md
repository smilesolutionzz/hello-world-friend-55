## 01. Email/Coaching/Unsubscribe 토큰 INSERT 무제한

- **OWASP** A01 Broken Access Control · **STRIDE** Tampering + Spoofing
- **Severity** High
- **Tables** `daily_coaching_email_tokens`, `email_unsubscribe_tokens`, `suppressed_emails`, `email_send_log`, `b2b_followup_queue`
- **Exploit**
  ```sql
  -- 로그인한 임의 사용자가 supabase-js로:
  await supabase.from('email_unsubscribe_tokens')
    .insert({ email: 'victim@x.com', token: 'attacker-known' });
  -- → /unsubscribe?token=attacker-known 호출 시 피해자가 모든 메일에서 차단됨.
  ```
- **Fix** 각 테이블에 `CREATE POLICY ... FOR INSERT TO authenticated WITH CHECK (false);` 추가 후 Edge Function에서만 `service_role`로 insert.

## 02. `consultation_reviews` 전체 인증 사용자 SELECT

- **OWASP** A01 BAC · **STRIDE** Information Disclosure
- **Severity** High
- **Table** `consultation_reviews` (policy: `USING (true)`)
- **Exploit** 어떤 로그인 사용자든 `select * from consultation_reviews` → 타 사용자의 자유 텍스트 리뷰(전문가 실명, 상담 내용 단서) 전량 열람.
- **Fix** `USING (user_id = auth.uid() OR expert_id IN (SELECT id FROM experts WHERE user_id = auth.uid()) OR is_public = true)`. `is_public` 컬럼 추가.

## 03. `expert_applications` 본인 SELECT 정책 부재

- **OWASP** A04 Insecure Design · **STRIDE** Repudiation
- **Severity** High
- **Table** `expert_applications` (이름·전화·생년월일·자격증 파일 PII)
- **Exploit** 신청자 본인이 자기 데이터를 못 봐서 클라이언트가 우회 경로(예: 다른 view, 캐시)로 PII를 가져올 가능성 → 향후 누군가 `USING (true)` SELECT 정책 추가 시 전원 PII 유출.
- **Fix** `CREATE POLICY ... FOR SELECT USING (auth.uid() = user_id);` 명시.

## 04. `institution_reviews` 가시성 플래그 부재

- **OWASP** A04 Insecure Design · **STRIDE** Information Disclosure
- **Severity** Medium
- **Fix** `is_public boolean default true` 추가, 정책에 `USING (is_public OR user_id = auth.uid())`.

## 05. SECURITY DEFINER 함수 anon/authenticated EXECUTE 허용 (대량)

- **OWASP** A01 BAC · **STRIDE** Elevation of Privilege
- **Severity** Medium (개별 함수 수동 점검 필요)
- **Detail** 다수 함수가 `SECURITY DEFINER`인데 anon/authenticated 모두에게 `EXECUTE`가 살아있음. 내부 트리거·헬퍼라면 권한 회수 필수.
- **Exploit** 익명 사용자가 `supabase.rpc('내부함수', { user_id: <피해자> })` 호출로 다른 사용자 데이터 변경.
- **Fix** 각 함수에 `REVOKE EXECUTE ON FUNCTION public.<fn>(...) FROM anon, authenticated;`. 진짜로 클라이언트가 호출해야 하는 함수만 화이트리스트.

## 06. Public Storage Buckets — 객체 리스팅 허용

- **OWASP** A05 Misconfiguration · **STRIDE** Information Disclosure
- **Severity** Medium (7건)
- **Exploit** `GET /storage/v1/object/list/<bucket>` 익명 호출 → 모든 파일 키 노출. 키를 알면 직접 다운로드.
- **Fix** 해당 bucket의 `storage.objects` SELECT 정책을 단일 객체 조회로 제한 (`storage.foldername(name)[1] = auth.uid()::text` 등).

## 07. `user_roles` 본인 SELECT 부재

- **OWASP** A04 Insecure Design · **STRIDE** Spoofing 방어 약화
- **Severity** Medium
- **Risk** 클라이언트가 role을 못 읽어서 localStorage/JWT claim에 캐시 → 변조 위험 ↑.
- **Fix** `CREATE POLICY ... FOR SELECT USING (auth.uid() = user_id);`

## 08. `b2b_partner_institutions.join_code` 노출

- **OWASP** A01 BAC · **STRIDE** Information Disclosure
- **Severity** Medium
- **Risk** 기관 owner SELECT 시 만료된 `join_code`도 함께 노출.
- **Fix** view 또는 column-level 권한으로 `join_code`/`join_code_expires_at` 마스킹, 발급 RPC 분리.

## 09. RLS Policy `USING (true)` (DML)

- **OWASP** A05 Misconfiguration · **STRIDE** Tampering
- **Severity** Medium (17건)
- **Action** 각 정책을 검토하여 SELECT 외 DML에서 `true`인 케이스 전부 사용자/관리자 스코프로 좁히기.

## 10. Auth — OTP 장기 만료

- **OWASP** A07 Identification & AuthN Failures
- **Severity** Low
- **Fix** Supabase Dashboard → Auth → OTP expiry ≤ 600초.

## 11. Auth — Leaked Password Protection 비활성

- **OWASP** A07 · **Severity** Low
- **Fix** Dashboard에서 활성화 (HaveIBeenPwned 연동).

## 12. Postgres 보안 패치 가용

- **OWASP** A06 Vulnerable Components · **Severity** Low
- **Fix** Supabase Dashboard → Database → Upgrade.
